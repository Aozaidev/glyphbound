# 第二波：战术地基

本文档是第二批实现规格。总纲见 `analysis.md`；第一批已实现规格见 `第一波-核心.md`。

当前状态：
- 第一批主干已实现：身命卷 14 字，`入/出/劫`，笔势，渡劫，魔杖升级，有效字注册。
- 第二批目标不是继续堆单体 buff，而是补齐“写给世界”的战术地基。
- 第二批只包含 7 个字：`印、山、泉、裂、壁、墨、界`。

---

## 设计目标

```text
1. 先落地施法目标协议，避免后续每个字返工。
2. 让铭刻阵（V 键）真正有价值：玩家可以改变战场，而不只是给自己加状态。
3. 让印成为方碑封印：先选方块，再把一个字写入区域规则场。
4. 墨/界提供区域规则，山河卷提供具体地形手段。
5. 当前版本仍不接抽卡；墨核只作为后续奖励货币预留。
```

---

## 施法目标协议

所有第二批字必须走统一上下文：

```text
CastContext
- caster：施法玩家
- staffTier：魔杖等级
- word：字
- targetType：SELF / ENTITY / BLOCK / AREA / MARK / EVENT
- origin：玩家位置
- targetPos：目标方块或区域中心
- sourceMark：若来自印，记录印来源
```

支持目标：

| 字 | 默认目标 | 可被印远程调用 | 说明 |
|---|---|---|---|
| 印 | BLOCK | 否 | 创建封印方碑 |
| 山 | BLOCK/AREA | 是 | 在目标处生成临时掩体 |
| 泉 | BLOCK/AREA | 是 | 在目标处生成恢复点 |
| 裂 | BLOCK/AREA | 是 | 在目标处生成裂隙/寻路干扰 |
| 壁 | BLOCK/AREA | 是 | 在目标处生成可破坏屏障 |
| 墨 | BLOCK/AREA | 是 | 生成墨染场 |
| 界 | BLOCK/AREA | 是 | 生成临时边界 |

第一批身命卷直接书写仍以 `SELF` 为主。进入 `印` 流程后，第二字会被转成方碑区域效果；不能在旧事件里另写一套目标逻辑。

### 底座通用能力

`aozainkmc` 只提供通用施写会话能力，不包含任何 Glyphbound 专用语义。

当前可用底座能力：

```text
InkMarkBeforeAttachEvent
- OCR 识别后、mark attach / 加笔势 / 扣耐久前触发
- 玩法模组可以取消普通施法
- 玩法模组可以请求客户端关闭阵、打开施写阵、打开铭刻阵、或进入选择方块状态

InkBlockTargetSelectedEvent
- 客户端进入选择方块状态后，玩家右键方块触发
- 服务端收到玩家、token、方块坐标
- 玩法模组可以继续请求客户端关闭阵、打开施写阵、打开铭刻阵、或再次进入选择方块状态

InkGlyphClientBehaviorRegistry
- 玩法模组注册哪些字识别成功后自动关闭当前书写阵
```

底座禁止出现 `印/入印/此字不可入印` 等玩法语义。`印` 的状态机、第二字是否可入印、空印清理都必须在 Glyphbound 内实现。

## 实现顺序

### 0. 地基：目标选择与封印流程

先实现通用目标选择能力：

```text
G/V 键打开书写阵
识别字
若玩法模组请求 BLOCK 目标，进入“选择目标方块”状态
玩家右键方块确认
服务端触发 InkBlockTargetSelectedEvent
离开法阵 10 格或切换维度取消
```

`印` 第一版先做两段式状态机，并接入 `净印`：
- 写 `印` 后进入选方块状态
- 右键方块后记录空印位置
- 底座自动重新打开施写阵
- 第二字为 `净` 时创建净印方碑
- 第二字为可入印但未接入的字时提示“X印尚未接入”
- 第二字不可入印时提示“此字不可入印”并清理
- 后续逐个字接入入印效果

### 1. 印：两段封印

定位：两段式施写协议——先选方碑再书字入印，把“近身书写”扩展为“区域规则场”。

**当前已实现：状态机骨架 + `净印`。其余可入印字先提示“X印尚未接入”。**

#### 流程

```text
1. 玩家写「印」→ 取消 mark attach → requestBlockTarget()
   客户端关闭法阵，进入方块选择模式
   服务端存 PendingSeal(phase=SELECT_BLOCK)

2. 玩家右键方块 → InkBlockTargetSelectedEvent
   验证 token、维度
   转换为 PendingSeal(phase=WAIT_SECOND_GLYPH, selectedBlock=pos, staffTier=写印时魔杖)
   requestOpenCastInput("印: 方块已选，请书写入印之字")
   客户端自动重新展开施写阵

3. 玩家写第二字 → InkMarkBeforeAttachEvent
   检测到 WAIT_SECOND_GLYPH → 取消 mark attach
   若第二字是「净」→ 创建净印方碑 → requestCloseInput("净印已成")
   若第二字可入印但未接入 → requestCloseInput("X印尚未接入")
   若第二字不可入印 → requestCloseInput("此字不可入印")
   最后销毁 PendingSeal
```

#### 当前入印效果：净印

```text
净印是持续性的净化方碑。
玩家进入净印范围时，清除 1 个负面效果。
同一玩家对同一净印 60 秒内只触发一次。
净印不处理墨伤，墨伤仍需要直接写「净」。
```

方碑规格取决于写「印」时的魔杖等级：

| 魔杖 | 半径 | 持续 |
|---|---:|---:|
| 木 | 5 格 | 3 分钟 |
| 石 | 7 格 | 4 分钟 |
| 铜 | 9 格 | 5 分 24 秒 |
| 铁 | 12 格 | 9 分钟 |
| 金 | 14 格 | 4 分 30 秒 |
| 钻石 | 18 格 | 12 分钟 |
| 下界合金 | 24 格 | 12 分钟 |

限制：
- 同一玩家同一维度最多存在 3 个印方碑，超过后最旧的消散。
- 方碑所在方块被挖掉后，封印立即终止。
- 玩家死亡、下线、离开维度或服务器停止时，自己的印方碑清理。
- 写「印」和写第二字都会正常消耗魔杖耐久并计入笔势。

#### 验证与打断

```text
- 第二字距方碑 >10 格 → "距封印方碑过远，封印中断" → 清理
- 玩家换维度 → "异维度，封印中断" → 清理
- 玩家死亡 → "封印中断" → 清理
- 玩家下线 → 静默清理
- 超时 60 秒 → "封印超时" → 清理
- 玩家重写「印」且处于 SELECT_BLOCK → 销毁旧状态，重新开始
- 服务端停机 → 静默清理（static Map clear）
```

#### 清理原则

所有 `PendingSeal` 存放在服务端 static `Map<UUID, PendingSeal>`，不跨存档、不跨会话。
死亡/下线/停机/超时/维度变更/超距均能触发清理，无残留。

#### 后续扩展

当具体字入印效果设计完成后：
- `handleSecondGlyph()` 改为查表判断第二字是否可入印
- 可入印 → 读取方碑坐标，创建区域效果，消耗方碑耐久/次数
- 不可入印 → 保持当前"此字不可入印"逻辑

## 山河卷

### 山：临时掩体

定位：制造掩体/阻挡/垫脚，改变战斗地形。

```text
木杖  低矮石垄，只挡箭矢和视线，不能挡怪物，持续 60 秒
铁杖  完整石墙，可阻挡怪物寻路和近战，持续 120 秒
钻石杖  石脊带荆棘，近战攻击石脊的怪物受微量魔法伤害，持续 180 秒
```

规则：
- 目标必须是可站立方块或其上方空气。
- 自适应地形：平地 2 格高墙，斜坡阶梯状屏障，水面生成石柱底座。
- 临时方块必须记录并回滚，不能覆盖容器、红石核心、传送门、基岩、黑曜石。

联动：
- 山后方站立时，`坚` 连续受击判定更宽松。
- `稳` 玩家站在山边缘不易被击退滑落。
- `界` 边界上的山更坚硬。

### 泉：移动恢复点

定位：战斗中可用的恢复源，与 `息` 互补。

```text
木杖  半径 3 格，每 5 秒恢复 0.5 饥饿，持续 30 秒
铁杖  半径 4 格，每 5 秒恢复 0.5 心和 0.5 饥饿，持续 60 秒
钻石杖  半径 5 格，额外清 1 个负面效果，短暂免疫中毒/凋零，持续 90 秒
```

规则：
- 同一玩家同时最多 2 个泉。
- 泉不能生成在空中、岩浆或不可替换方块上。
- 怪物会优先攻击泉；泉被破坏时溅射半径 3 格恢复 2 心。

联动：
- `息` 期间靠近泉会打断息，但泉恢复量翻倍。
- `龟` 可保护泉。
- 墨染场内泉变墨泉：恢复翻倍，但给予 5 秒缓慢 I。

### 裂：地面裂隙

定位：改变寻路，制造陷阱。

```text
木杖  细微裂痕，怪物寻路绕开，持续 45 秒
铁杖  裂隙喷墨烟，穿过获得 3 秒失明，持续 60 秒
钻石杖  裂隙扩展至多 3 格，边缘减速 -20%，持续 90 秒
```

规则：
- 书写后 2 秒生效，给玩家离开时间。
- 硬质方块上延迟加倍。
- 不能破坏或替换基岩、黑曜石、容器、命令方块。

联动：
- `稳` 玩家不受裂隙减速。
- `界` 内裂隙更容易扩展。
- 墨染场内同一玩家的裂隙会自动连成网状。

### 壁：垂直屏障

定位：短时间隔断路线，保护后排。

```text
木杖  3 高 × 2 宽，20 点结构生命，持续 30 秒
铁杖  5 高 × 3 宽，40 点结构生命，持续 60 秒
钻石杖  6 高 × 4 宽，80 点结构生命，带荆棘反弹，持续 90 秒
```

规则：
- 必须贴地生成，不能悬空。
- 怪物攻击和爆炸会削减结构生命。
- 生命归零或到期后回滚，播放碎石粒子。

联动：
- 与 `山` 形成“前墙后脊”阵地。
- 与 `盾/龟` 叠加为防线。
- `界` 边界上的壁更坚硬。

## 界律卷

### 墨：墨染场

定位：风险与收益放大器，区域规则地基。

```text
木杖  半径 8 格，持续 2 分钟
铁杖  半径 32 格，持续 5 分钟
钻石杖  半径 64 格，持续 10 分钟
```

场内规则：
- 心额外 +10 心上限，不与心自身上限叠加。
- 怒积累速度 +20%，冲击波范围 +1。
- 忍转移比例 +10%，但结算间隔缩短 25%。
- 怪物生成率 +50%，怪物主动朝中心移动。
- 写其他字额外消耗 20% 耐久。
- 死亡怪物掉落翻倍。

墨核规则：
- 当前第二批先不接墨核实物。
- 墨核作为后续唯一新增奖励货币预留。
- 等墨染场稳定后再加入：普通怪低概率、精英高概率、首领必掉。

清理：
- 同一玩家同一维度只能维持 1 个墨染场。
- 后写 `墨` 刷新中心与持续时间，不叠加。
- 玩家死亡/退出清理自己的墨染场。

### 界：临时边界

定位：战术隔离，封闭战场。

```text
木杖  半径 6 格，普通怪物不能穿越，持续 30 秒
铁杖  半径 10 格，精英穿越受 4 心伤害并减速，投射物减速 50%，持续 45 秒
钻石杖  半径 15 格，首领穿越受 8 心伤害并眩晕 2 秒，持续 60 秒
```

规则：
- 界内玩家不能写 `墨`，防止规则套娃。
- 界消失时，界内怪物仇恨重置到最近玩家。
- 两个界重叠时，后写覆盖先写。

联动：
- `山/壁` 写在界边界上会更坚硬。
- `裂` 在界内更容易扩展。
- 墨染场内写界生成墨界壁，边界触碰造成微量魔法伤害。

## 与第一批的关系

```text
入/劫：提供压力和成长闭环
印：把“近身写世界”扩展成“远程布置”
山/泉/裂/壁：给坚/稳/息/隐/怒提供战术场景
墨/界：把第一批身体字放进更高风险的区域规则里验证
```

测试重点：
- `心/坚/稳/忍/净/怒` 在墨染场内是否仍可控。
- `息` 与 `泉` 的互斥是否好懂。
- `隐` 是否会被灯轨迹、墨染场、精英追踪形成取舍。
- `山/壁/界` 是否能改变怪物移动，而不是只做装饰。

## 收口标准

第二批可以收口的条件：

1. 铭刻阵能选择方块，并能取消/超距自动关闭。
2. 印能创建、显示、到期、被破坏清理，并能被支持 MARK 的字消耗。
3. 山/泉/裂/壁都能生成、记录、到期回滚，不破坏关键方块。
4. 墨/界都能创建区域规则，并在死亡/退出/切维度时清理。
5. 第二批所有字都通过有效字注册，未实装字仍提示“此字未通”。
6. Patchouli 指南更新到第二批，不再保留旧 12 字说明。
